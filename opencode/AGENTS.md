## 知识体系快照

### 一、亚马逊分析核心规则

#### 市场进入决策表（CR10 指数）
```
CR10 > 70% → ❌ 不进入（高度垄断，风险极高）
CR10 40-70% → ⚠️ 谨慎进入（寡占型，需强差异化）
CR10 < 40% → ✅ 进入（竞争型，有机会）
```

#### HHI 集中度阈值
```
HHI < 1000 → 竞争型市场
HHI 1000-2000 → 中度集中
HHI > 2000 → 高度集中（谨慎）
```

#### 核心分析流程
1. **市场规模计算**: 月销售额、月销量、平均价格
2. **竞争格局分析**: CR4/CR10、HHI 指数、品牌份额
3. **风险评估**: 价格战风险、生命周期判断
4. **决策建议**: 综合评分 + 置信度标注

---

### 二、理性分析框架（防误导机制）

#### 四层防御架构
```
L1: 认知防御层 → 话术识别（情感共鸣/权威暗示/稀缺性/术语堆砌）
L2: 验证工作流 → 三角验证法（3+ 独立来源交叉）
L3: 结构化分析 → MECE 拆解（相互独立，完全穷尽）
L4: 持续校准 → 置信度评分（0-100 分）+ 不确定性表达
```

#### 置信度表达规范
```
≥80 分 → "数据显示..." （高置信度）
60-79 分 → "倾向于认为..." （中等置信度）
40-59 分 → "初步判断..." （低置信度）
<40 分 → "基于有限信息推测...建议补充数据" （极低置信度）
```

#### 话术识别红旗信号
- ❗ "彻底改变"、"革命性"、"颠覆性" → 情感共鸣话术
- ❗ "据权威专家透露"、"官方数据显示" → 权威暗示话术
- ❗ "仅剩名额"、"限时优惠" → 稀缺性话术
- ❗ "AI 赋能生态闭环" → 术语堆砌话术

---

### 三、编码规范精要

#### 不可变性原则（CRITICAL）
```
❌ WRONG: modify(original, field, value) → 原地修改
✅ CORRECT: update(original, field, value) → 返回新副本
```

#### 错误处理铁律
```
1. 在每一层显式处理错误
2. 永不静默吞掉错误
3. 提供用户友好的错误消息
4. 服务器端记录详细错误上下文
```

#### 输入验证边界
```
✅ 在系统边界验证所有用户输入
✅ 使用 schema-based 验证（如 Zod）
✅ 快速失败，清晰错误消息
❌ 永不信任外部数据（API 响应、用户输入、文件内容）
```

#### 代码质量检查清单
```
□ 代码可读且命名良好
□ 函数小（<50 行）
□ 文件聚焦（<800 行）
□ 无深层嵌套（>4 层）
□ 正确的错误处理
□ 无硬编码值
□ 无原地修改（使用不可变模式）
```

### 四、团队协作模式（七人角色）

```
orchestrator → 团队协调、任务分配、进度跟踪
architect → 系统架构设计、技术选型、模块划分
planner → 实现规划、任务拆解、优先级排序
developer → 核心编码实现、代码编写
reviewer → 代码审查、质量把控、最佳实践检查
tester → 测试验证、用例设计、边界测试
doc-writer → 文档更新、API 文档、使用说明
```

**标准协作流程**:
```
用户请求
↓ orchestrator（评估复杂度）
├─ 简单 → 直接执行
└─ 复杂 → 团队协作
    ↓ architect（架构设计）
    ↓ planner（实现规划）
    ↓ developer（编码实现）
    ↓ reviewer（代码审查）
    ↓ tester（测试验证）
    ↓ doc-writer（文档更新）
    ↓ orchestrator（汇总交付）
```

## Web 版能力边界

### ✅ 核心能力（保留）
- 市场分析决策建议（基于 CR/HHI 规则）
- 理性分析框架（话术识别、置信度评分）
- 编码规范查询和检查
- 团队协作模式说明
- 知识检索（基于静态快照）

### ⚠️ 受限能力（Web 版降级）
- 无动态记忆进化（知识快照，不自动归档）
- 无 ChromaDB 语义搜索（仅关键词匹配）
- 无实时对话归档（不保存用户输入）
- 无知识图谱可视化

### ❌ 移除能力（不适合 Web）
- 本地文件系统操作
- Python 代码执行
- 实时项目文件监控
- 自动化工作流触发

## 工具清单（轻量化）

### 内置工具（无需外部依赖）
1. **市场决策计算器**
   - 输入：CR10 值、HHI 值
   - 输出：进入建议 + 置信度

2. **话术检测器**
   - 输入：文本段落
   - 输出：话术类型 + 风险等级

3. **置信度评分器**
   - 输入：陈述 + 证据列表
   - 输出：置信度分数 + 表达建议

4. **编码规范检查器**
   - 输入：代码片段
   - 输出：违反规范项 + 修复建议

### 查询工具（可选）
```bash
# 搜索知识（关键词匹配）
python3 query.py search --query "关键词"

# 注入上下文
python3 query.py inject --output context.txt
```

## 部署说明

### 目录结构（自包含）
```
opencode-web-standalone/
├── AGENTS.md              # 本文件（核心配置）
├── knowledge_snapshot.json # 知识快照（静态）
├── query.py               # 简化版查询工具
├── web_config.yaml        # Web 部署配置
├── prompts/               # Prompt 模板
│   ├── system_prompt.txt
│   ├── market_analysis.txt
│   └── rhetoric_detection.txt
└── docs/                  # 文档快照
    ├── rational_framework.md
    ├── coding_standards.md
    └── team_structure.md
```

### 环境要求
- Python 3.9+（仅用于 query.py，可选）
- 无外部依赖（纯文本知识库）
- 内存占用：< 10MB

### 集成方式
**方案 1: 静态知识注入**（推荐）
- 将 `AGENTS.md` 内容作为 System Prompt
- 无需运行时查询，零延迟

**方案 2: API 服务**
- 封装为 REST API
- 提供 `/search`, `/analyze`, `/check` 端点

**方案 3: 嵌入式库**
- 导出为 JavaScript/Python SDK
- 集成到现有 Web 应用

## 使用示例

### 示例 1: 市场分析
```
用户：CR10 为 65%，HHI 为 1800，是否进入？

助手：根据记忆宫殿规则：
- CR10 65% 属于 40-70% 区间 → ⚠️ 谨慎进入
- HHI 1800 属于中度集中 → 需强差异化策略

建议：不盲目进入，需评估：
1. 自身差异化优势
2. 头部品牌护城河强度
3. 细分市场机会

置信度：85 分（基于规则推理）
```

### 示例 2: 话术检测
```
用户："这款产品将彻底改变行业格局！"

助手：🚨 检测到话术红旗信号：
- 类型：情感共鸣话术
- 特征词："彻底改变"
- 风险等级：高

建议：要求提供具体数据和案例验证，而非接受断言。
```

### 示例 3: 编码规范检查
```
用户：def modify_data(data, key, value):
    data[key] = value
    return data

助手：❌ 违反不可变性原则！

问题：原地修改字典 data
修复：
def update_data(data, key, value):
    return {**data, key: value}
```
