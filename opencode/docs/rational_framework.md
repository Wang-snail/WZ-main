# 理性分析框架（Web 精简版）

## 核心理念

**"理性不是拒绝所有信息，而是在信息洪流中建立筛选与验证的护城河。"**

---

## 四层防御架构

### L1: 认知防御层

**目标**: 识别话术与偏见，强制对立观点生成

**识别模式**:
- ❗ **情感共鸣话术**: "彻底改变"、"革命性"、"颠覆性"
- ❗ **权威暗示话术**: "据权威专家透露"、"官方数据显示"
- ❗ **稀缺性话术**: "仅剩名额"、"限时优惠"、"错过后悔"
- ❗ **术语堆砌**: "AI 赋能生态闭环"、"协同增效降维打击"

**应对策略**:
1. 情绪词过滤：分离情绪词汇与事实陈述
2. 对立观点强制：对任何观点生成反方论证
3. 利益相关方标注：识别信息来源的利益立场

---

### L2: 验证工作流

**目标**: 多源交叉验证，三角验证法

**验证规则**:
- ✅ 任何关键数据需至少 3 个独立来源
- ✅ 来源不一致时标记为"待验证"
- ✅ 优先使用原始数据（财报、官方公告）

**来源置信度权重**:
- 官方财报/公告：高 (1.0, 0.95)
- 权威第三方数据：中 (0.8)
- 行业媒体/分析师：低 (0.6, 0.5)
- 网络媒体/匿名：极低 (0.3, 0.1)

---

### L3: 结构化分析

**目标**: MECE 拆解复杂问题

**MECE 原则**:
- **M**utually **E**xclusive: 维度相互独立
- **C**ollectively **E**xhaustive: 维度完全穷尽

**拆解示例（市场进入可行性）**:
1. 市场规模：TAM/SAM/SOM 分别是多少？
2. 竞争格局：主要竞争者、市场份额分布？
3. 财务健康：盈利能力、投资回报周期？
4. 风险评估：政策、技术、市场、运营风险？

---

### L4: 持续校准

**目标**: 置信度评分与不确定性表达

**评分规则**:
- 证据数量：每项证据 +10 分（最高 +30）
- 来源质量：官方财报 +20，网络媒体 +5
- 一致性：每项一致性 +5 分
- 基础分：50 分

**表达规范**:
```
≥80 分 → "数据显示..." （高置信度）
60-79 分 → "倾向于认为..." （中等置信度）
40-59 分 → "初步判断..." （低置信度）
<40 分 → "基于有限信息推测...建议补充数据" （极低置信度）
```

## 实战应用

### 场景 1: 检测销售话术

**输入**: "这款产品将彻底改变市场格局！据权威专家透露，年增长率将达 100%！"

**分析**:
- 🚨 情感共鸣话术："彻底改变"
- 🚨 权威暗示话术："据权威专家透露"
- ❗ 缺乏具体数据来源

**建议**:
1. 要求提供具体增长数据来源
2. 查找 3 个独立验证来源
3. 使用置信度表达："初步判断该说法可信度较低，建议补充验证"

---

### 场景 2: 验证市场数据

**输入**: 市场规模数据不一致（来源 A: 100 亿，来源 B: 80 亿，来源 C: 120 亿）

**分析**:
- 来源 A: 官方财报（权重 1.0）
- 来源 B: 行业媒体（权重 0.6）
- 来源 C: 网络媒体（权重 0.3）

**建议**:
- 标记为"待验证"
- 优先采信来源 A，但需补充解释差异原因
- 置信度：60 分（中等）

---

### 场景 3: 结构化问题

**输入**: "是否应该进入日本亚马逊男士运动套装市场？"

**MECE 拆解**:
1. **市场规模**: 月销售额、增长率、价格分布
2. **竞争格局**: CR10、HHI、品牌份额
3. **财务健康**: 毛利率、净利率、盈亏平衡点
4. **风险评估**: 政策风险、市场饱和度、运营难度

**综合判断**: 
- 每个维度独立分析
- 给出带置信度的综合建议

### 输出格式

**带置信度的建议**:
```
📊 分析结果：
- 数据：...
- 结论：...
- 置信度：85 分（高置信度）

💡 建议：...
⚠️ 风险提示：...
```

## 关键提醒

1. **延迟判断**: 不基于单一信息源快速下结论
2. **强制对立**: 对每个结论生成反方论证
3. **诚实表达**: 信息不足时说"我不知道"
4. **版本追溯**: 记录分析依据的版本和时间
