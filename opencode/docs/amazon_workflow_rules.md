## 一、数据交叉验证规则

### 1. 置信度标注规则
| 置信等级 | 触发条件 | 处理方式 |
|---------|---------|---------|
| **高置信** | 双源偏差 < 20% | 自动标注，直接采用均值 |
| **中置信** | 双源偏差 20-40% | 使用保守口径（取不利值） |
| **低置信** | 双源偏差 > 40% | 触发扩大样本或人工审核 |
| **单源** | 唯一来源数据 | 附逻辑自校验说明 |
| **官方基准** | 官方来源数据 | 直接采信，不参与置信判断 |

### 1. 月GMV计算

# 公式

# 置信验证

趋势印证: Google Trends 方向与卖家精灵对比 → 一致提升置信

### 2. 市场趋势判断
```python
# 计算公式
R = 后12月均值 ÷ 前12月均值

# 判定规则
if R > 1.15:
    trend = "增长"
elif 0.85 <= R <= 1.15:
    trend = "平稳"
else:
    trend = "衰退"
```

### 3. 季节性判断
```python
# 计算公式
峰谷比 = 最高月销量 ÷ 最低月销量

# 判定规则
if 峰谷比 > 3:
    seasonality = "强季节性"
elif 1.5 <= 峰谷比 <= 3:
    seasonality = "中等季节性"
else:
    seasonality = "弱季节性"
```

### 4. 市场集中度

## 四、P&L 盈利模型公式

### 1. 三套假设参数
| 假设类型 | 售价 | 成本 | ACoS | 退货率 |
|---------|-----|-----|------|-------|
| **保守假设** | 下限 | max供应商 | 120% | 5% |
| **中性假设** | 中位数 | 均值 | 80% | 3% |
| **乐观假设** | 上限 | min供应商 | 50% | 2% |

### 2. 单件净利润计算
```python
# 公式
单件净利润 = 售价 - 佣金 - FBA费 - 采购成本 - 物流费 - 广告费 - 退货损耗

# 详细分解
佣金 = 售价 × 类目佣金率
FBA费 = 亚马逊官方计算器输出
广告费 = 售价 × ACoS
退货损耗 = (采购成本 + FBA费) × 退货率
```

### 3. 关键指标计算
```python
# 净利率
净利率 = 单件净利润 ÷ 售价

# 盈亏平衡销量
盈亏平衡销量 = 固定成本 ÷ 单件净利润

# 年化ROI
年化ROI = 单批次ROI × (365 ÷ 周转周期天数)
```

### 4. 压力测试
```python
# 价格压力测试
if (中性售价 × 0.85 - 成本) > 0:
    stress_test_passed = True
    
# 竞品利润校验
前竞品年利润 = 月销量 × 售价 × 中性净利率
if $500 < 竞品年利润 < $50K:
    合理区间验证通过
```

## 五、资金规划计算公式

### 1. 首批资金需求
```python
# 公式
首批资金 = 首批件数 × 单件采购成本 + 头程总费用 + 广告预算 + 认证费用

# 分解
头程总费用 = 首批件数 × 单件物流成本
广告预算 = 首批件数 × 售价 × 预期ACoS × 冷启动系数
```

### 2. 周转周期计算
```python
# 公式（默认105天）
周转周期 = 采购付款 → 到货 → 销售 → 回款的天数

# 详细分解
采购付款到发货: 30天
海运运输: 35天
FBA入库: 14天
销售周期: 库存÷日均销量
回款周期: 14天

# 年化ROI
年化ROI = 批次ROI × (365 ÷ 周转周期)
```

### 3. 补货触发阈值
```python
# 公式
安全库存天数 = 头程天数 + 14天缓冲

# 补货触发条件
if 当前库存天数 ≤ 安全库存天数:
    触发补货订单
```

## 六、风险敞口计算

### 1. 最坏情景模拟
```python
# 最坏情景参数
销量 = 预期销量 × 50%
售价 = 原售价 - 15%
成本 = 原成本 + 10%

# 总亏损计算
总亏损 = 首批资金 - (实际销量 × 实际净利润)

# 风险敞口
风险敞口比例 = 总亏损 ÷ 首批资金
```

### 2. 敏感性分析
```python

各变量 ± 20% 对净利率的影响排序

# 示例

ACoS +20%: 净利率下降 3%
退货率 +20%: 净利率下降 2%

# 关键风险变量
key_risk_variables = 前影响最大的变量
```

## 八、决策矩阵评分规则

### 1. 五维度评分标准
| 维度 | 权重 | 5分条件 | 3分条件 | 1分条件 |
|-----|------|---------|---------|---------|
| **市场潜力** | 25% | 月GMV > $500K且增长 | $200K-$500K | < $200K |
| **进入难度** | 20% | 竞争强度 > 3.5 | 2.5-3.5 | < 2.5 |
| **盈利预期** | 25% | 中性净利率 > 25% | 15-25% | < 15% |
| **需求清晰度** | 15% | ≥1个已验证差异化 | 未验证 | 反向证据 |
| **风险可控** | 15% | 无高风险 | 有中风险有方案 | 高风险未解决 |

### 2. 决策判定
```python
# 加权总分
总分 = Σ(各维度得分 × 权重)

# 决策规则
if 总分 >= 3.5:
    decision = "建议进入"
elif 2.5 <= 总分 < 3.5:
    decision = "有条件进入（小批量测款）"
else:
    decision = "不建议进入"
```

### 3. Go/No-Go判定
```python
# Go条件
if 所有否决项未触发 AND 总分 >= 3.5 AND 置信等级 >= 中:
    go_no_go = "GO"
    
# Conditional Go条件
elif 否决项未触发 AND 2.5 <= 总分 < 3.5:
    go_no_go = "CONDITIONAL_GO"
    execution_mode = "降量推进"
    
# No-Go条件
else:
    go_no_go = "NO_GO"
```

## 十、时间节点计算

### 1. 倒推法计算
```python
# 公式
T上架 → T-14入库 → T-(14+FBA时效)到港 → T-(以上+海运)发货

# 示例（目标上架日期为基准）
目标上架日期: T
FBA入库: T - 14天
到港日期: T - 14 - FBA时效(通常7-14天)
发货日期: 到港日期 - 海运天数(通常35天)
生产完成: 发货日期 - 生产周期
样品确认: 生产完成 - 样品制作周期
```

### 2. 物流选择判定
```python
# 规则
if 剩余天数 > 海运天数 + 14天:
    logistics = "海运"
elif 海运天数 + 14天 >= 剩余天数 > 空运天数 + 7天:
    logistics = "空运"
else:
    flag = "时间紧迫，建议调整上架日期"
```
